# ML Data Preparation Agent Report

Date: 2026-04-25

## Existing Implementation Audit

- `scripts/assemble_dataset.py` was already the canonical CLI for building `data/processed/master_dataset.csv`, `data/processed/model_matrix.csv`, and internal templates.
- `app/services/dataset_assembly.py` already owned source normalization, canonical schema alignment, source provenance, and row-level deduplication through `assemble_master_dataset()`.
- Existing default behavior included the local mock training records plus available RepoRT bulk and MCMRT processed sources.
- METLIN Figshare and Kaggle METLIN descriptor-aligned CSVs were both present locally, but there was no profile/source policy preventing accidental target-row double counting.

## Changed Files

- `app/services/dataset_assembly.py`
  - Added source registry/profile resolution for `default`, `expanded_ml`, and `explicit`.
  - Added explicit sidecar/exclusion policy for Kaggle METLIN descriptor-aligned exports.
  - Added markdown summary report writer for selected, excluded, and sidecar row counts.
- `scripts/assemble_dataset.py`
  - Added `--source-profile` and `--reports-dir`.
  - Routed external source selection through the service resolver.
  - Preserved explicit `--additional-source` behavior as the `explicit` profile.
  - Removed redundant pre-normalization before `assemble_master_dataset()`.
- `tests/test_dataset_assembly.py`
  - Added tests for `expanded_ml` profile source selection.
  - Added script-level test for explicit METLIN Figshare plus Kaggle sidecar exclusion and report output.
- `reports/dataset_assembly_summary.md`
  - Regenerated by the default assembly command.
- `reports/ml_data_preparation_summary.md`
  - Added compact expanded-profile source row count and duplicate/sidecar policy summary.

## Known Limitations

- Full `expanded_ml` model-matrix assembly over ReTiNA + MCMRT + RepoRT + METLIN Figshare was attempted but timed out after 15 minutes before writing expanded master/model outputs. The likely bottleneck is RDKit descriptor generation over roughly 214k target input rows.
- The Kaggle descriptor sidecar is retained and reported, but descriptor joins are not implemented in this slice.
- The duplicate policy prevents known METLIN Figshare/Kaggle double counting at source-selection time; deeper cross-source chemical/method duplicates still rely on the existing canonical dedupe key.
