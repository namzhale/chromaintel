# ChromaIntel Artifact Policy

The repository keeps source code, fixtures, small reports, and reproducible metadata in git. Large generated datasets and trained model bundles are local build artifacts unless an explicit artifact store or Git LFS policy is introduced.

## Do Not Commit By Default

These files are generated by expanded data assembly and training and can exceed normal GitHub limits:

- `data/processed/master_dataset.csv`
- `data/processed/model_matrix.csv`
- `data/processed/models/trained_forward_bundle.joblib`
- `data/processed/dl/graph_manifest.csv`
- `data/processed/dl/smiles_transformer_manifest.csv`
- large public source snapshots under `data/raw/public_sources/`

Before every commit, run:

```powershell
git diff --cached --name-only
```

If any large generated artifact appears in the staged list, unstage it unless the current task explicitly introduces Git LFS or another reviewed artifact workflow.

## Reproducible Local Build

Expanded local data can be recreated with:

```powershell
.\.venv\Scripts\python.exe scripts\fetch_public_datasets.py --retina --rows 0 --output-name retina_hf_full
.\.venv\Scripts\python.exe scripts\fetch_public_datasets.py --metlin-smrt-figshare --rows 0 --output-name metlin_smrt_figshare
.\.venv\Scripts\python.exe scripts\fetch_public_datasets.py --bulk-report --target-rows 5000 --max-datasets 80 --output-name report_bulk_5k
.\.venv\Scripts\python.exe scripts\fetch_public_datasets.py --mcmrt --output-name mcmrt_supplement
.\.venv\Scripts\python.exe scripts\assemble_dataset.py --source-profile expanded_ml
.\.venv\Scripts\python.exe scripts\train_models.py --feature-set core --quick
.\.venv\Scripts\python.exe scripts\prepare_dl_datasets.py
```

The current expanded local build produced 213,941 canonical rows and a 43-column model matrix.

## Recommended Future Artifact Store

For handoff or CI, use one of these reviewed approaches:

- GitHub Releases for versioned demo bundles.
- Git LFS for small-to-medium binary artifacts if repository quotas are acceptable.
- An external object store for full public-data snapshots and trained models.

Each released artifact should include:

- source manifest and source licenses
- command used to build it
- row counts and hash checksums
- model metrics report
- compatibility note for the code commit that produced it
